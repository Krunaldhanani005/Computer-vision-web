Face Recognition

Identity detection and matching pipeline

Active

How it works

Each video frame is scaled and run through an SSD face detector. Detected face regions are passed to dlib to extract 128-dimensional feature vectors. These are compared against stored encodings using Euclidean distance (threshold 0.5). Identities are classified as known, blacklist, or unknown.

Model Details

DetectorOpenCV DNN (SSD)
Encoder128-d face_recognition vectors
ZoneMongoDB — polygon points
FPS~15 (every 2nd frame)

Inputs & Outputs

InputWebcam / RTSP stream
OutputAnnotated MJPEG + labels
LimitationsRequires frontal face, good lighting
Restricted Area Monitoring

Zone-based intrusion detection with ArcFace recognition

Active

How it works

YUNet detects faces; ArcFace extracts 512-d embeddings compared to a registered persons database. Every 3rd frame is processed with a 10-second cooldown per track. Unknown faces inside the polygon zone trigger an alert with a cropped snapshot saved to static/alerts/ and logged to MongoDB.

Model Details

Person detectYOLOv8 nano
Face detectYuNet (OpenCV)
Encoding matchArcFace — threshold 0.45
FPS~10 (every 3rd frame)

Inputs & Outputs

InputWebcam / RTSP stream
Alertsrestricted_area_db.alerts
Snapshotsstatic/alerts/*.jpg
Object Detection

YOLOv8 real-time object classification

Active

How it works

Every 2nd frame is passed to YOLOv8n/s. Detections are filtered to four target classes. Bounding boxes with class labels and confidence percentages are drawn and streamed as MJPEG. No zone or database dependency — fully stateless.

Model Details

ArchitectureYOLOv8
Classesbottle · chair · watch · phone
OutputBounding boxes + confidence
FPS~15–20 (every 2nd frame)
